# Python for ML
> Python is the lingua franca of machine learning — every major framework (scikit-learn, PyTorch,
> TensorFlow) speaks it, and its data stack (NumPy, Pandas, Matplotlib) is where the work happens.
> You don't need to be a software engineer; you need the core language plus comfort with the data
> tools. This card gets you the Python foundation; the next three cards cover the data stack.

**Why it matters:** you can't do ML without writing code, and that code is Python. Interviewers
expect fluency with the basics — lists/dicts, comprehensions, functions, slicing, and reading data —
because every take-home and live-coding ML exercise is written in it.

**⭐ Start here — suggested path:**

1. **Learn the core language** — watch [Python Full Course for Beginners](https://www.youtube.com/watch?v=rfscVS0vtbw) (or do the [official tutorial](https://docs.python.org/3/tutorial/)). *Variables, control flow, functions, lists/dicts — the foundation.*
2. **Practice interactively** — work through [Kaggle Learn: Python](https://www.kaggle.com/learn/python). *Short, hands-on lessons that stick because you type them.*
3. **Go straight to the ML-relevant bits** — watch [Python for Data Science (Pandas/NumPy/Matplotlib)](https://www.youtube.com/watch?v=LHBE6Q9XlzI). *The slice of Python you actually use in ML, in one course.*
4. **Build the right habits** — read [Automate the Boring Stuff (Ch. 1–6)](https://automatetheboringstuff.com/). *Free book; turns syntax into the ability to actually get things done.*
5. **Keep a reference open** — bookmark [the Python docs](https://docs.python.org/3/) and [W3Schools Python](https://www.w3schools.com/python/). *Look things up as you go rather than memorizing.*
