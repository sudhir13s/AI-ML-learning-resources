# arXiv & Papers with Code — Finding & Tracking Research
> Where ML research actually lives before (and often instead of) peer-reviewed venues, and the tools
> that turn the firehose into a tracked, searchable, code-linked stream. arXiv is the preprint
> archive; Papers with Code links papers to runnable implementations and leaderboards; a small set
> of discovery tools (Semantic Scholar, Connected Papers, HF Daily Papers) keep you on top of it.

**Why it matters:** "how do you find the SOTA method for X and a reference implementation?" is a
real workflow question. Knowing arXiv categories, how to use Papers with Code's SOTA tables, and
how to expand a citation graph is the difference between rediscovering a method and reusing one.

**⭐ Start here — suggested path:**

1. **Learn the arXiv map** — skim [arXiv help](https://info.arxiv.org/help/) and bookmark [cs.LG (ML)](https://arxiv.org/list/cs.LG/recent), [cs.CL (NLP)](https://arxiv.org/list/cs.CL/recent), [cs.CV (vision)](https://arxiv.org/list/cs.CV/recent). *Knowing which category a topic lives in is how you browse the right firehose instead of all of it.*
2. **Use Papers with Code for "what works now"** — open a ⭐ [SOTA leaderboard](https://paperswithcode.com/sota) for a task you care about and click through to a paper + its code. *Leaderboards answer "what's the best method on this benchmark, and can I run it?" in one place.*
3. **Track top papers daily** — follow [Hugging Face Daily Papers](https://huggingface.co/papers); skim the top 5 each morning. *Community upvotes pre-triage the firehose so you read signal, not volume.*
4. **Expand from one paper** — drop a paper into [Connected Papers](https://www.connectedpapers.com/) or [Semantic Scholar](https://www.semanticscholar.org/) to find its ancestors, descendants, and most-cited neighbors. *Citation graphs reveal the lineage and the papers everyone builds on.*
5. **Read the PDF efficiently** — use [alphaXiv](https://www.alphaxiv.org/) or the [Semantic Reader](https://www.semanticscholar.org/product/semantic-reader) for inline references and discussion. *Combine with the three-pass method from card 01.*

## 📄 Key Papers
- [arXiv.org e-Print archive](https://arxiv.org/) — **Cornell** — the archive itself; browse [cs.LG](https://arxiv.org/list/cs.LG/recent), [cs.CL](https://arxiv.org/list/cs.CL/recent), [cs.CV](https://arxiv.org/list/cs.CV/recent), [stat.ML](https://arxiv.org/list/stat.ML/recent).
- [Construction of the Literature Graph in Semantic Scholar](https://arxiv.org/abs/1805.02262) — **Ammar et al. (2018)** — how a citation/knowledge graph for papers is built (the engine behind discovery tools).
- [SPECTER: Document-level Representation Learning for Scientific Papers](https://arxiv.org/abs/2004.07180) — **Cohan et al. (2020)** — paper embeddings that power "find similar papers" recommendations.
